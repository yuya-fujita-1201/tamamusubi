## 1. 戦闘モデル

### 1.1 基本方式

見下ろし型リアルタイムアクション（ARPG）。マップ上に敵が配置され、接触／索敵半径（`sightR`）に入ると敵がアクティブ化。プレイヤーは自由移動しながら武器攻撃・スキル・回避を行う。

- **フレームレート目標**: 60 fps 基準。フレーム単位の当たり判定。
- **ターゲティング**: 自動ロック非採用（ロックオン無し）。攻撃方向＝プレイヤーの向き。敵密集時は武器のリーチ・範囲で複数を巻き込む設計。
- **被弾**: ノックバック（KB）は被弾方向の反対に一定距離押し出す。距離は攻撃側の重さによって小/中/大。無敵フレーム（I-frame）あり（被弾後 20F 目安）。

### 1.2 フレームデータ思想

「テレグラフ（予備動作）が必ず見える設計」を徹底する。敵の `telegraphF`（かまえF数）はプレイヤーが回避行動を取れる最低ラインに設定する。

- **かまえ（準備F）**: 攻撃に入る前の予備動作。敵・武器ともに必ず置く。
- **判定（ヒット判定F）**: 攻撃が実際に当たる区間。
- **HS（ヒットストップ）**: 命中時、攻撃側と被弾側の両方が一瞬停止するフレーム。重さの「手触り」を作る核。
- **後隙（硬直F）**: 攻撃後に次の行動ができない区間。長いほどリスク高。
- **役割**: かまえを短くするほど素早い武器、後隙を長くするほど重くリターンが大きい武器として差別化する。

### 1.3 ヒットストップ方針

| 武器カテゴリ | HS目安 | 理由 |
|---|---|---|
| 木槌（単発重打） | 8F | 重さの体感を作る核心 |
| 祓い刀（連撃） | 3F×hit | 連撃テンポを崩さない範囲で手応え |
| 御鏡（継続照射） | 0F（per-tick） | 継続ダメージは停止なしで光の圧力を演出 |
| 舞扇（多段機動） | 2F×hit | 移動撃ちを妨げない最小HS |
| 薙刀（貫通） | 4F | 多体ヒット時は全対象に同時適用 |
| 和弓（単発精密） | 5F | 単発着弾の満足感 |
| 式符（式鬼） | 2F | 自律打撃・爆ぜは別扱い |
| 錫杖（AoE） | 4F | 突き鳴らしの衝撃感 |

## 2. ダメージ計算式方針

### 2.1 基本方針

ダメージは**方向ごとに2本**定義する（与＝プレイヤー→敵 ／ 被＝敵→プレイヤー）。両式とも最低保証 1。

```
与ダメージ（プレイヤー→敵） = (プレイヤー攻撃力 × 武器倍率 × 属性ボーナス) − 敵防御値
被ダメージ（敵→プレイヤー） = 敵attack − プレイヤー防御値（衣）
最低保証ダメージ = 1（どちらの式も最終値が1を下回らない）
```

- **プレイヤー攻撃力**: 装備武器の段階（g1〜g10）とレベルで決まる基礎値。数値インフレなし。
- **武器倍率**: 武器系統の g1 = 1.00 を基準に g10 で概ね 1.45〜1.60 倍。
- **属性ボーナス**: 弱点一致で ×1.5〜2.0（弱点でなければ ×1.0。ペナルティなし）。与ダメージ側のみに掛かる。
- **敵防御値**: 与ダメージから減算する敵側の硬さ。**現行実装 `src/data/enemies.ts` の `Enemy` 型に `def` フィールドは無く、`tank` アーキタイプは `deflectSword`（正面防御フラグ）で硬さを表現している**。本式では「敵防御値」を採用するか `deflectSword` 方式を継続するかは Wave 2 実装スパイクで確定する（採用する場合 `enemies.ts` に `def?` を追加。当面は 敵防御値=0 として与ダメージ＝攻撃力×倍率×属性 で運用可）。
- **プレイヤー防御値**: 装備「衣」の防御値。被ダメージから減算。現行実装 `src/scenes/field.ts` は `player.takeDamage(e.def.atk, ...)` と敵 attack を直渡ししており、衣防御値の減算（`src/state/game.ts` への防御値保持）は実装フェーズで追加する。

> 旧版は「被ダメージ＝(攻撃力×武器倍率×属性)−防御値」と1本の式に与/被を混同し、攻撃力・防御値とも*プレイヤー自身*の値を充てていた（自分の攻撃力から自分の防御を引く論理破綻）。本版で方向別2本へ分離し、実装（enemies.ts に def 無し・敵 atk 直渡し）と整合させた。

### 2.2 会心

- 会心確率はデフォルト 5%。勾玉（例:「HP満タン時のみ会心2倍」など）や衣で変動。
- 会心時ダメージ = 通常ダメージ × 1.5（固定）。

### 2.3 倍率の実数値について

各武器・スキルの具体的係数は各フェーズのプロトタイプ実装で実測・調整する（§11 参照）。本版は計算式の**方針と構造**を定めるに留める。

## 4. 祓い波システム

### 4.1 ゲージ蓄積条件

| 行動 | 蓄積量 | 備考 |
|---|---|---|
| 通常攻撃（命中） | 小 | 武器系統によらず蓄積 |
| 縛り祓い（W1チャージ・命中） | 中～大 | 結び師の家業。序盤の主要蓄積源 |
| 被弾 | 極小 | 防御で粘るプレイにも微量 |
| 錫杖 響鳴（skl_007のチャージ技） | — | ゲージ蓄積速度2倍（3秒間） |
| skl_006 共鳴の勾玉（パッシブ） | — | 所持勾玉数に応じてゲージ最大量・蓄積速度が上昇 |
| skl_014 鼓舞の舞（神楽） | — | 蓄積速度を一定時間上昇（バフ） |

### 4.2 浄化技（祓い波の使用）

- ゲージが一定量以上蓄積した状態で祓い波ボタンを入力すると発動。
- 基本形: skl_001「祓い波・初伝」——前方〜自身周囲へ波動。穢れの妖に高ダメージ＋浄化演出（光になって還る）。
- ゲージが少ない場合は不発（消費ゼロ）。ゲージが溜まるほど同技の威力が増す（量的段階）。

### 4.3 第二形態（常世祓い波）解放トリガー

- **物語進行でのみ解放**（武器・レベルは無関係）。
- 常世到達後、npc_004「常世の巫」から「世界の歌の記憶」を授かることで skl_005「祓い波・第二形態」が解放される。
- 威力・浄化量が上昇し、演出が「散る → 昇る（魂が常世へ還る）」へ変化。光の色も変化。

### 4.4 勾玉による性質変化系列

「祓い波変化系」カテゴリの勾玉（mag_###）を装備することで祓い波に性質を付与できる。

| 性質の例 | 効果イメージ |
|---|---|
| 属性化 | 祓い波に光/風/炎などの属性を付与（対応する弱点の敵に×1.5〜2.0） |
| 回復付与 | 祓い波命中時に自身HPを微回復 |
| 範囲化 | 祓い波の射程・範囲を拡張 |
| 連続 | ゲージ消費量を減らし連続発動を可能にする |

詳細は EQUIP_SYSTEM_DESIGN の勾玉スロット仕様を参照。
