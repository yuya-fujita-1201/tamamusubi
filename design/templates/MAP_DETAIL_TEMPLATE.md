## 10. map-quality パイプライン接続（必須）

### 10.1 map_goal 入力（レビュー用 目的文）

> `quality/MAP_REVIEW_PROMPT.md` と map-art-reviewer が読む「このマップが達成すべき目的」。1〜3文で、主役・高さ・歩行・世界観を明記する。

```
<例: 始まりの里山郷の入口。南の河口(H0)から棚田の段(H1)を登り、北の鎮守の杜の社(H2)へ一本道で通じる。
主役は社の高台、背景は棚田。プレイヤーが最初に「八洲の里山」の空気を掴む導入マップ。>
```

### 10.2 heightmap 入力（`checker/heightmaps/<map_id>.json` の設計）

> `levels[y][x] = 0(H0)/1(H1)/2(H2)`、`w*h` と一致。装飾前に塗る。下記は設計意図のスケッチ（実装時に正寸へ展開）。

```jsonc
{
  "map": "<map_id>",
  "w": <W>, "h": <H>,
  // levels[y][x]: 0=H0低地 / 1=H1中段 / 2=H2高所
  "levels": [
    [ /* y=0 の各x */ ]
  ]
}
```

### 10.3 DoD 5本柱への対応メモ

MAP_DEFINITION_OF_DONE（`../../../quality/MAP_DEFINITION_OF_DONE.md`）の各柱を、このマップでどう満たすか1行ずつ：

- **構造**: <H0/H1/H2 が全て存在し境界に段差。>
- **視認性**: <25%縮小でも主役と地形が読める。>
- **歩行範囲**: <導線が途切れず通る。>
- **主役と背景**: <主役＝…／背景＝…の優先順位。>
- **タイル境界**: <境界処理・ノイズを抑える方針。>
