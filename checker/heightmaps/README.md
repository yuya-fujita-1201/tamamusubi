# heightmaps/ — 宣言的な高さマップ（任意）

GPT Pro 制作フロー Step 2「装飾の前に H0/H1/H2 を先に決める」を機械検査に繋ぐためのファイル。

`checker/heightmaps/<map>.json` を置くと、`HEIGHT` チェックが
**全ての高低差境界（隣接セルでレベルが違う箇所）に段差表現（擁壁/土手/崖/階段/滝/橋/川）があるか**
を宣言的に検査する。無い場合は「段差規律（擁壁前面の真下に影）」のみ検査し、INFO で宣言を促す。

## 形式

```jsonc
{
  "map": "tanada",
  "w": 56, "h": 46,
  // levels[y][x] = 0(H0低地) / 1(H1中段) / 2(H2高所)
  "levels": [
    [2,2,2, ...],   // y=0 の各 x
    ...
  ]
}
```

あるいは flat 配列でも可:

```jsonc
{ "map": "tanada", "w": 56, "h": 46, "grid": [2,2,2, ... /* w*h 個 */] }
```

## 作り方（推奨）

1. マップの目的を決めたら、**装飾を置く前に**高さを塗る（紙でもスプレッドシートでも可）。
2. 高所(神域)=2、中段(道・棚田・テラス)=1、低地(川・入口)=0。
3. レベルが切り替わる境界に、必ず段差素材（`kaIshigaki`/`kaDote`/`kaShrineWall`/階段/滝/橋/川）を置く。
4. `checker/run_check.sh <map>` で境界の取りこぼしを検出。

> 高さは「タイルの見た目」ではなく「設計意図」。先に決めることで、AIが装飾で誤魔化す前に構造を固定できる。

`_template.json` を雛形として複製して使う。
