### 1.2 近接の棲み分け（ターゲット数×状況の4軸）

| 武器 | 軸 |
|---|---|

遠隔は 御鏡（継続照射・魔法）と和弓（単発精密・物理）で操作性が完全に異なる。式符＝設置、錫杖＝支援で全8種の役割が直交する。

### 1.3 設計原則（v1 から継続）

- **全武器が終盤まで現役**: 数値インフレなし。強化は鍛冶（銭＋特産）。差は役割の直交性で作る
- **弱点相性はボーナス型**（どの武器でも倒せる・正解で×1.5〜2.0）
- **段階導入**: 体験版3種→地域スプリントごとに1〜2種。新武器は入手地域に専用ギミック＋専用の敵をセット実装
- **命名規則**: プレイヤー向け呼称は3〜4音まで（御鏡・式符・木槌…）。神話由来のフルネーム（八咫の欠片 等）はフレーバーテキストへ。実在神名は技名にも使わない（×天照放射 → ○日輪解放）

### 1.4 体験版の武器導線（Phase 1A/1B の段階化——GPTレビュー反映）

```
Phase 1A（最小縦切り・15〜20分）: 祓い刀（初期・縛り祓い）→ 木槌（里山の宝箱）
Phase 1B（公開体験版・30〜45分）: ＋ 御鏡（根の洞の宝箱）
```

- 1A は武器2種で「移動・探索・戦闘・祓い波・勾玉・ヒビ岩の再訪体験」を完結させる（武器理解の時間を確保）
- 1B で遠隔（御鏡）と光ギミックを追加し、公開クオリティに引き上げる

- 里山に**白ヒビ岩を3〜4ヶ所**配置（木槌入手前に必ず1つ視界に入れる→入手後の再訪体験）
- 根の洞は岩割り＋曇り鏡石の光ギミックで「武器で道が開く」を凝縮体験。**暗所攻略が御鏡頼みにならないよう、提灯を里の店で购入可能にしておく**（鍵化防止）
- 体験版で「縛り祓い」を使う場面を序盤チュートリアルに1回組み込む（結び師の家業の体感）

## 2. フィールドギミック原則

> **武器ギミックは「鍵」にしない。「ご褒美と近道」にする。**

1. **メインルート非依存**: 各地域のボスへの道は武器ギミックなしで到達可能。**マップ実装のDoDに「ギミック無視でボス到達可能」「縄矢の橋には必ず別ルートが存在」チェックを明記**
2. **ギミックの先**: レア勾玉／衣／手控え帳ページ／隠し祠（依代の恵み）／ショートカット
3. **視覚記号は1武器1種に統一**（§1.1表）。見た瞬間に必要武器が分かり「後で来よう」と記憶に残る
4. **1ギミック＝1オブジェクト種**（Asset Forge 生成1回の粒度）。複合パズル（複数武器連携）は高天原の隠し祠限定
5. **密度**: 秘密の密度ルール（3画面に1個上限）とギミック配置は共有カウント。**未入手武器の記号は同一マップに2種まで**（「後で来よう」の記憶負荷を抑える）

## 4. 勾玉（厳選の核）

- スロット3。特性カテゴリ: 攻撃系／守り系／移動系／**祓い波変化系**（属性化・回復付与・範囲化）／探索系（採集・感知・銭運）
- ユニーク＝固定配置（危険地帯・隠し祠・スニーク先取り）。「変な効果」歓迎（例:「被弾してもノックバックしない代わりに防御-2」「夜だけ攻撃+3」「HP満タン時のみ会心2倍」）
- 体験版: 固定5〜8個（うち1個はスニーク先取り）。製品版フェーズ1以降: 地方特性付き汎用勾玉の低確率ドロップ追加
- ※アクセサリー語彙（勾玉・御守り・数珠・鈴・根付・組紐・御札）のうち「御札」はアクセ専用。武器の「式符」とは別物（命名競合解消済み）

## 6. 成長システム（最小構成）

- レベル制（ゆるやか・上限低め）。主役は勾玉・衣・武器の使い分け
- 武器強化: 鍛冶屋（銭＋特産）各2〜3段階
- 依代の恵み: 隠し祠の恒久バフ（HP上限+等）＝探索報酬の最上位
- 祓い波: 物語進行で第二形態（常世）。性質変化は勾玉で

## 7. 整合・Phase 0 への申し送り

### 物語との接続（v2 更新）
- **縛り祓い（祓い刀チャージ）＝結び師の家業の象徴**（ムチ廃止の補填。開始3分から「結び師の子」が戦闘で見える）
- **式符の式鬼＝白く淡い光の玉**。終盤シロと再会した時に「式符で呼んでいたのはシロのような存在だったのか」と無言で繋がる（Story.md の式鳥と接続）
- 式符（武器・設置持続・常用）と三流派「式」スキル（一発大技・MP消費・習得制）は操作レイヤーで分担。物語上は「父・廻人が式の技を旅の護符に落とし込んだもの」
- 依代の注連縄を張り直す「結びの修繕」は武器でなくフィールドのインタラクト行動として検討（依代が灯る演出と接続）

### Phase 0 / 各フェーズ着手時の検証項目（PLAN §10 に追加）
1. **御鏡の照射**（体験版武器）: 構え状態＋直線ビーム判定＋曇り鏡石の反射判定が既存エンジンで成立するか。二段階式（移動中減衰）の実装可否
2. **木槌の重打フレーム**（体験版武器）: 新規フレームデータの手触り検証（戦闘コア移植と同時）
3. 式符の式鬼FSM（フェーズ3着手時）: 自律移動・同時2体・爆ぜの最小実装スパイク
4. 舞扇の**移動撃ちステート**（フェーズ3着手時）: 「攻撃中移動不可」前提の状態機械への影響をスパイクで確認
5. ギミック「鍵化禁止」チェックをマップ実装DoDに組み込む（§2-1）

### 新規アセット・フレームデータが必要な武器
- 新規フレームデータ: 木槌・御鏡（照射）・和弓・式符・舞扇（5種）／流用ベース: 祓い刀（剣）・薙刀（槍）・錫杖（杖＋突き鳴らし追加）
